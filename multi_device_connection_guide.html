C++ 멀티 디바이스(다중 장비) 접속 및 제어 가이드

본 문서는 C++ 개발자가 본 라이브러리(CommunicationWrapper.dll)를 사용하여 두 대 이상의 Power PMAC 장비에 동시에 접속하고 제어하는 방법에 집중하여 설명합니다.

1. 다중 장비 접속(Connect) 원리

본 라이브러리는 PmacManagerWrapper 단일 매니저 클래스를 통해 내부적으로 IP 주소를 키(Key)로 하는 딕셔너리(Dictionary) 형태로 다수의 장비 통신 객체를 관리합니다. 따라서 개발자는 접속할 장비의 IP만 넘겨주면 독립된 통신 객체를 획득할 수 있습니다.

접속 예제 코드 (C++)

// 1. 매니저 객체 선언 (일반적으로 다이얼로그 클래스의 멤버로 선언)
PmacManagerWrapper^ m_manager = gcnew PmacManagerWrapper();

// 2. IP 문자열 정의
System::String^ ipA = gcnew System::String("192.168.0.200");
System::String^ ipB = gcnew System::String("192.168.0.201");

// 3. GetDevice(IP)를 호출하여 각 장비의 전용 객체 포인터 획득
auto devA = m_manager->GetDevice(ipA);
auto devB = m_manager->GetDevice(ipB);

// 4. 각각의 객체로 물리적 접속 시도
bool isConnectedA = devA->Connect(ipA);
bool isConnectedB = devB->Connect(ipB);

if (isConnectedA && isConnectedB) {
    // 다중 장비 동시 접속 완료
}

2. 다중 장비 개별 제어 (Command)

접속이 완료되면, 앞서 획득한 devA와 devB 포인터를 사용하여 완전히 격리된 명령을 동시에 전송할 수 있습니다. 각 포인터는 독립된 SSH 소켓과 스레드 동기화 락(Lock)을 소유하고 있으므로 패킷이 엉키지 않습니다.

명령어 전송 예제 (C++)

// PMAC A 장비(192.168.0.200)의 1번 모터 조그(JOG) 구동
devA->SendCommand("#1J+");

// PMAC B 장비(192.168.0.201)의 1번 모터 조그(JOG) 구동
devB->SendCommand("#1J+");

// PMAC A 장비 정지
devA->SendCommand("#1J/");

3. 실시간 동시 모니터링 (타이머 기반)

여러 장비의 상태(위치값 등)를 동시에 모니터링할 때는 C++의 타이머(OnTimer)나 별도의 쓰레드를 사용하여 각 기기 객체의 GetResponse를 연속 호출하면 됩니다.

위치값(ActPos) 병렬 폴링 예제 (C++)

void CPDKSampleUIDlg::OnTimer(UINT_PTR nIDEvent)
{
    // 타이머 틱(예: 200ms)마다 양쪽 장비에 상태를 요청
    
    // 1. PMAC A의 위치 쿼리
    auto devA = m_manager->GetDevice(gcnew System::String("192.168.0.200"));
    if (devA->IsConnected()) 
    {
        CString resA = devA->GetResponse("Motor[1].Pos");
        // 응답 예시: "Motor[1].Pos=123.456\r\n\x06" -> '=' 뒷부분 파싱 필요
    }

    // 2. PMAC B의 위치 쿼리
    auto devB = m_manager->GetDevice(gcnew System::String("192.168.0.201"));
    if (devB->IsConnected()) 
    {
        CString resB = devB->GetResponse("Motor[1].Pos");
    }

    CDialogEx::OnTimer(nIDEvent);
}
[중요 주의사항]
devA->IsConnected() 처럼 통신 연결 상태를 먼저 체크한 후 GetResponse를 호출해야 합니다. 장비가 꺼져있을 때 호출하면 통신 라이브러리 내부에서 예기치 않은 에러(Null 예외)가 발생할 수 있습니다.
